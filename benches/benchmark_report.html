Abstract

This report presents a controlled performance comparison between x1zzLang, a domain-specific compiled language built atop Rust and the Polars LazyFrame execution engine, and Python Pandas, the de-facto standard for tabular data processing. Benchmarks are conducted across three dataset scales (Small ≈ 228K rows, Medium ≈ 1.6M rows, Large ≈ 3.4M rows) derived from six real-world annual air-quality measurement files. All pipeline stages are identical across engines. At the Large scale x1zzLang achieves a 3.84× speedup and a 3308 MB reduction in peak resident-set memory relative to Pandas.
