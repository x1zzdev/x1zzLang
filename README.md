# x1zzLang

> *"데이터 분석을 위한 도구가 아니라, 데이터 분석 자체가 문법인 언어"*

**겉은 스크립트, 속은 컴파일.**

x1zzLang은 고성능 데이터 분석을 위해 설계된 새로운 프로그래밍 언어입니다.  
파일 확장자는 `.xzz`, 철학은 단 하나 — 복잡함은 컴파일러가, 단순함은 사용자에게.

---

## 왜 x1zzLang인가

현재 데이터 분석 생태계는 구조적으로 불편하다.

Python으로 분석을 하려면 `pandas`, `numpy`, `matplotlib`을 각각 import하고,
버전 충돌을 관리하며, 런타임에서야 타입 오류를 마주친다.
성능이 필요할 때는 또 다른 레이어가 쌓인다.

x1zzLang은 이 구조를 언어 설계 수준에서 뒤집는다.

- **라이브러리가 없다.** `filter`, `groupBy`, `mean`, `plot` — 이것들이 이미 문법이다.
- **런타임 오류가 없다.** 스키마 기반 타입 시스템이 데이터 오류를 컴파일 시점에 잡는다.
- **느리지 않다.** 내부적으로 Rust + Polars로 트랜스파일되어 네이티브 수준으로 실행된다.

---

## 코드가 어떻게 생겼냐면
```xzz
v data = load("sales.csv") :: SalesSchema;

data
  |> filter(col("price") > 100)
  |> groupBy("region")
  |> mean("price")
  |> plot.bar(x: "region", y: "price");
```
x1zz run analysis.xzz

`import`도 없고, `main()`도 없다.  
파일 첫 줄부터 바로 분석이다. 나머지는 컴파일러가 책임진다.

---

## 핵심 아이디어

**`|>` 파이프라인 연산자**  
데이터가 변환 단계를 흐르듯 통과한다. 중첩 함수 호출 없이, 읽히는 순서대로 실행된다.

**Safe-Load 시스템**  
데이터를 로드할 때 스키마를 선언하면, 컬럼 존재 여부와 타입을 컴파일 시점에 검증한다.
조용히 생성되는 `NaN`은 x1zzLang에서 허용되지 않는다.

**스크립트 지향 실행 모델**  
엔트리 포인트가 없다. Python처럼 쓰고, 컴파일 언어처럼 실행된다.

**Zero-cost Abstraction**  
파이프라인의 가독성은 성능 저하로 이어지지 않는다.
추상화 비용은 0이다 — 문법이 곧 최적화된 Rust 코드로 변환되기 때문에.
